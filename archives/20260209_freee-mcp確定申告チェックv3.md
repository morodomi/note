# Claude Codeで確定申告の漏れを見つけた話

タグ候補: #ClaudeCode #確定申告 #freee #MCP #個人事業主 #AI #フリーランス #帳簿

確定申告の時期です。

普段からClaude Codeばかり使っている。コードも書かせるし、ドキュメントも任せる。だったら帳簿チェックも手伝ってもらえばいい。

ノリでfreeeのデータを読めるMCPサーバーを作った。2日でできた。結果、入れ忘れが大量に見つかった。

---

## 何が面倒だったか

個人事業主の確定申告は、売上より経費のチェックが厄介だ。

売上はそんなにたくさんないから、確認は簡単。問題は経費。毎月のサーバ費は入っているか。通信費は漏れていないか。未決済の取引が残っていないか。

一番気になるのが、「入れ忘れている経費がないか」。

税理士も人間だ。帳簿に入っていない項目はチェックのしようがない。でも、前年や前月と比較すれば「あれ、去年はあった通信費が今年はゼロだぞ」と気づける。その比較作業を、freeeの画面で手動でやっていた。

正直、これは人間がやる仕事じゃない。だったらClaude Codeにやらせよう。

---

## 作ったもの

2つ作った。

**freee-mcp**: freee会計のAPIをClaude Codeから叩けるようにするMCPサーバー。MCPはClaude Codeに外部ツールを接続する仕組みで、これを使えばfreeeのデータをClaude Codeから直接参照できる。収支サマリー、科目別集計、月次推移、未決済取引の一覧が取得できる。読み取り専用。

**tax-advisor**: 3つの専門エージェントが並行で帳簿をレビューするスキル。

- P/L分析: 前年比較で大きな変動を検出、月次の異常検知
- 取引チェック: 未決済取引、期間外取引、大額取引の検出
- コンプライアンス: 按分計上のリマインド、要注意科目の警告

Claude Codeで「帳簿をレビューして」と言えば、3エージェントが並行で走って結果を返してくれる。

---

## なぜ読み取り専用にしたか

freeeのAPIには書き込み系のエンドポイントもある。取引の登録、更新、削除。でも、今回はGETだけMCP化した。

AIが勝手に帳簿を書き換えたら怖い。

レビューして指摘するだけなら、最悪間違っていても実害はない。「この経費、去年より増えてますよ」と言われて、自分で確認すればいい。でも「この取引、間違ってるので修正しておきました」は困る。

AIには読ませる。判断は人間がする。この線引きは意識した。

---

## 実際に使ってみた

入れ忘れが大量に見つかった。

サーバ費、携帯の通信費。去年は毎月入っていたのに、今年はゼロになっている科目がいくつもあった。まだ入力していなかったものもあれば、完全に存在を忘れていた経費もあった。比較してもらうと一目瞭然だった。

今、請求書をダウンロードして1つずつ帳簿を付けている。

家事按分の相談もした。去年から変えていないが、「この按分率で合っているか」を聞けるのは安心感がある。

---

## 2日で作れた理由

読み取り専用だからシンプルだった。freeeのデータを読んで、分析して、レポートする。書き込みはしない。APIの設計が単純になる。

freeeのDeveloperサイトのドキュメントをAIに読ませて、「自分が確定申告前にやっているチェックを自動化したい」と伝えた。何をチェックしたいかは自分の話なので、要件が明確だった。

freeeはOAuth認証に対応しているので、パスワードをアプリに渡さずに済む。セキュリティ的に安心だった。使いながら多少の調整はした。APIがpartner_idしか返さないので、partner_nameを取得するように修正したり。でも、大きなハマりどころはなかった。

---

## 限界

正直に書く。

**比較対象がないと見つからない**。去年と今年を比較して「去年はあったのに今年はない」で気づける。でも、2025年から新しく始めた取引で、途中の月に1件も入力していなければ、比較対象がない。見つけようがない。

**数百円の取引は埋もれる**。大きな変動は検出できるが、細かい取引の漏れは気づきにくい。

**万能ではない**。目視チェックで去年と金額比較すれば、ある程度は自分でも見つけられたと思う。ただ、全科目を手作業で比較するのは面倒だし、見落とす。そこをAIが補ってくれる。

---

## 使い方

リポジトリは公開している。

https://github.com/morodomi/freee-mcp

セットアップの流れ:

1. freee Developersサイトでアプリ登録（Client ID / Secretを取得）
2. `uv sync` でインストール
3. `.env` に認証情報を設定
4. `claude mcp add freee-mcp -- uv run --directory /path/to/freee-mcp freee-mcp` でClaude Codeに登録
5. OAuth認証を通す

tax-advisorスキルは、Claude Codeのスキルとしてインストールすれば使える。

Claude Codeを使っている人なら、自分でも作れると思う。freeeのAPIドキュメントを読ませて、「帳簿チェックしたい」と伝えれば、似たようなものはできる。
