# Claude Codeに褒められたけど提案は全却下した

期待していたのは、Claude Code標準ツールの使い分けだった。WebSearchとcurlはどう使い分けるべきか、WebFetchとBashのcurlで何が違うのか。そういう実践的なアドバイスが欲しかった。

X.comで `/insights` コマンドが紹介されているのを見かけた。使い方を分析してアドバイスをくれるらしい。もしかしたら、そういう実践的な情報ももらえるかもしれない。CLAUDE.mdをもっと良くできると考え、試してみた。

結果、褒めてはもらえた。ただ、提案は1つも採用しなかった。どんな内容だったか見てみよう。

## insightsコマンドとは

Claude Codeで `/insights` を実行すると、自分の使用履歴を分析してレポートを生成してくれる機能。

出力は `~/.claude/usage-data/report.html` に保存される。ブラウザで見やすいHTML形式で確認できる。

## レポートで出力された内容

私の場合、約50日間で67,391メッセージ、6,474セッションという使用状況がレポートに出力された。2月4日に試したので、12月分からだと思う。

### 評価された内容

レポートには「印象的な成果」として、以下の内容が出てきた。

- TDDワークフローの規律正しい実行
- マルチエージェントオーケストレーションの活用
- 130以上のテストを維持する品質管理
- Bash優先の自動化哲学（Bashコマンドがファイル編集の約2倍）

85%という高い完了成功率も表示された。

### 改善提案

「試してみるべきこと」として、次のようなレコメンドが出てきた。

- TDDワークフローフェーズをエンコードするカスタムスキルを作成する
- 編集後にテストスイートを自動実行するフックを設定する
- plan-reviewでは5エージェントを使うようCLAUDE.mdに記載する

## 良かった点

自分の使い方を客観的に見られるのは面白い。

Claude Code自体がまだ新しいツールで、みんなが正しい使い方を模索している段階だ。そんな中で、公式から「ある程度正しく使えてますよ」とお墨付きをもらえるのは、自分の使い方が間違っていなかったと確認できる。

「Bashコマンドがファイル編集の約2倍」という数字も、確かにそうだろうなと思いつつ、具体的な比率を見ると「そんなに差があるのか」と気づきがあった。数字で可視化されると、自分の傾向が明確になる。

## 物足りなかった点

正直なところ、すでにClaude Codeを使い込んでいる人には物足りない。

**1. セッションによって結果が偏る**

`~/.claude/projects/` 内の全セッションが対象になるが、実行毎に処理されるセッション数には上限がある。また、短時間のセッションは除外される。

私の場合、普段はPHPとPythonが多い。しかしたまたまFlutterのセッションが多く処理された。そのせいで「Game Development with Flutter」が主要プロジェクトとして認識されていた。

**2. 提案が「すでにやってること」ばかり**

TDDスキル化のレコメンドがあったが、すでに `tdd-core` として実装済み。CLAUDE.mdへの追加項目も、ほとんど導入済みの内容だった。

**3. 実践的なアドバイスがない**

冒頭で書いた通り、期待していたのはこういうアドバイスだった。

- WebSearchとcurlはどう使い分けるべきか
- WebFetchとBashのcurlで何が違うのか
- このシーンではこちらが向いている

レコメンドはあくまで「こうした方がいい」という抽象的なものが多く、「こう使うと速い」という実践知は得られなかった。

## 誰に向いているか

Claude Codeを使い始めて数週間から数ヶ月の中級者に向いていると思う。

初心者だとデータが少なすぎてレポートが浅くなる。逆にヘビーユーザーだと、レコメンドの多くが「すでにやってる」になる。

使うとしたら、週1回ぐらいのペースで実行して、CLAUDE.mdの見直しや、提案されたスキルを試してみるのが良さそう。

## まとめ

insightsコマンドは、自分のClaude Code使用状況を可視化してくれる。

公式から「よく使えてる」とお墨付きをもらえた点は収穫だった。新しいツールで正解がわからない中、自分のやり方が間違っていなかったと確認できるのは安心感がある。

ただ、レコメンドはすべて「すでに実装済み」か「Claude Codeと相談して受け入れない判断をしたもの」だった。結局、受け入れた提案は無かった。残念。

でも、自分の使い方を客観視する機会としては悪くない。今回はたまたまObsidianのvaultで実行したので、別のプロジェクトのClaude Codeで試すと違う結果が出るかもしれない。
