# コード0行でCLI完成。Claude Codeとの1時間の開発記録

Backlogでチケットを振るのが面倒だった。

Web UIを開いて、プロジェクトを選んで、課題を作成して、担当者を選んで、優先度を設定して。1件ならいい。でも5件、10件と振るときに、毎回同じ画面遷移を繰り返すのが嫌になった。

CLIがあれば一発なのに。Backlog公式のCLIは存在しない。サードパーティ製もピンとこない。じゃあ作るか。ただし、自分でコードは書かない。Claude Codeに全部やらせる。

## planモードで対話した

Claude Codeにはplanモードがある。「こういうツールを作りたい」と伝えると、実装計画を立ててくれる機能だ。

まずBacklog APIのリファレンスURLを渡した。「npmとGitHubで既存のBacklog CLIがあるか調べて」。

AIの報告。npmにbacklog-cliパッケージはあるが公式ではない。Backlog APIは公開されていて、backlog-jsというSDKがある。

pnpmで自作する。CLIフレームワークはCommander、npm globalインストール前提。方針を伝えると、計画が返ってきた。8サイクル。

- Cycle 0: プロジェクトの雛形作成
- Cycle 1: 設定管理（認証情報の保存）
- Cycle 2: APIクライアント生成
- Cycle 3: 認証コマンド
- Cycle 4: プロジェクト一覧
- Cycle 5: 課題一覧（フィルタ付き）
- Cycle 6: 課題詳細表示
- Cycle 7: CLI統合

計画を見て、承認した。

## AIが勝手にAPIを叩き始めた

最初はAIにBacklogを操作させるのが怖かった。参照系だけなら安全だろうと思って、課題の一覧と詳細表示だけ作らせた。

15分ほど使っていた。完了チェックやコメント追加くらいはAIにやらせたかったが、CLIにその機能がない。セキュリティチェックで脆弱性を検出するたびにBacklogチケットを手で起こす作業が始まると、コピペの量が爆発した。CLIで課題を検索して、ブラウザに切り替えて新規チケットを作って、またCLIに戻って。

そのうちClaude Codeが気づいた。backlog-cliにcreateコマンドがないことに。

次にAIがやったことに冷や汗が出た。backlog-cliのディレクトリに移動してソースコードを読んだ。設定ファイルからAPI KEYの保存場所を特定した。そしてcurlでBacklog APIを直接叩き始めた。

動いてしまった。

createコマンドを追加することにした。4サイクルの計画を立てさせて、7分で完了。ついでにAPI KEYをローテーションした。

全12サイクル。書き込み系は機能が多く、テストも一気に増えて140テスト、カバレッジ98.96%。トータル1時間。

## CLIで何ができるようになったか

```
backlog issue create -p PROJECT_KEY \
  --summary "認証機能の実装" \
  --type "タスク" \
  --priority "中" \
```

これでチケットが作れる。Web UIで5回クリックしていた操作が1コマンドになった。誰に振るとか、マイルストーンとかは後でWeb UIでちょっと修正する。

コメント追加もCLIから。ステータス更新もCLIから。チケットを振る作業が確実に楽になった。

Claude Codeに使わせるときは `backlog --help` を実行させて、コマンド体系を把握させた。「チケット管理に使うからCLAUDE.mdに書いておいて」と伝えたら、使い方を自分で記録してくれた。assigneeの指定はまだうまくいかないことがある。Web UIで修正するのが面倒になったらCLIを修正する。

GitHub Issueで十分じゃないか。チームにエンジニアしかいなければそうする。でもエンジニア以外のメンバーもいるし、ガントチャートはやはりメンバーには貴重だったりする。現場のツールを変えるより、自分の作業だけCLIで効率化するほうが現実的だ。

## コード0行の正体

「コード0行」は事実だ。TypeScript、Commander.js、backlog-js。全部Claude Codeが選定して、全部Claude Codeが書いた。自分はplanモードで方針を決めて、計画を承認して、完成したCLIを動かしただけ。

ただ、CLIの設計パターンを知っているから計画の良し悪しを判断できた。Service層を分離してAPIクライアントを差し替え可能にしている構造を見て、テストしやすいと判断した。セキュリティの確認ポイントがどこかわかるのも、経験があるからだ。AIが勝手にcurlでAPIを叩いたときに「まずい」と思えたのも同じ理由だ。

コードを1行も書かずに、140テスト・99%カバレッジの実用ツールが1時間で完成した。AIが書いたコードの品質を、TDDが担保する。テストが通っていれば、人間が書いたかAIが書いたかは関係ない。
