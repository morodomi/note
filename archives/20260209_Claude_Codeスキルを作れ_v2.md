# Claude Codeスキルを作って、自分専用のAIにする

タグ候補: #ClaudeCode #スキル #AI #開発 #カスタマイズ

Claude Codeを使っているなら、スキルを作れ。

毎回同じプロンプトを打つ手間が消える。使い込むほど、自分専用のClaude Codeになる。

---

## 毎回プロンプトを打っていた

自分のTDD（テスト駆動開発）サイクルはtdd-skillsというスキル集で自動化している。サイクル中は問題ない。問題は、サイクルが終わった後だ。

「次、何やるんだっけ」が分からなくなる。

以前は毎回こう打っていた。

「issue確認して、cycle docs確認して、最近のcommit見て、次のタスク候補をリストアップして」

面倒だった。毎回同じプロンプト。しかも確認先を1つ忘れると、タスクの抜け漏れが起きる。

サイクル中は自動化されているのに、サイクル間のつなぎが手作業だった。

だからスキルにした。

---

## スキルとは何か

Claude Codeには「スキル」という拡張機能がある。やりたい作業手順をマークダウンで書いておくと、スラッシュコマンドや自然言語の指示で呼び出せる。

実体はSKILL.mdというファイル1つ。この「次、何やるんだっけ」を解決するために作ったsearch-taskは、こう書いた。

```markdown
---
name: search-task
description: |
  commit, issue, cycle docsを確認して次のタスクを決定するスキル。
  使用するタイミング:
  (1) 「次のタスク」「次何する」「search-task」
  (2) 「タスクを探して」「次の作業」
  (3) 「何をやるべき」「優先タスク」
allowed-tools: Bash, Read, Glob, Grep
---

# Search Task - 次のタスク決定スキル

commit履歴、GitHub Issues、cycle docsを横断的に確認し、
次に着手すべきタスクを特定する。

## 実行手順

以下の3つの情報源を並行で収集し、総合的に次のタスクを判断する。

### Step 1: 情報収集（並行実行）

#### 1-A. 直近のcommit履歴を確認
git log --oneline -20
最近何が完了したかを把握する。

#### 1-B. GitHub Issuesを確認
未対応のissueが次のタスク候補になる。

#### 1-C. Cycle Docsを確認
各cycle docのStatusフィールドを確認。
DONE以外のステータスがあれば、途中のサイクルがある。

#### 1-D. STATUS.mdを確認
完了済み、進行中、未着手のタスクを把握。

### Step 2: 分析・判断

1. 途中のTDDサイクル（最優先）
2. Open Issues
3. STATUS.mdのTODO
4. 新規タスクの提案

### Step 3: レポート出力
推奨タスク、理由、対応するIssue、TDDフェーズを報告する。
```

日本語で「何を確認して、どう判断して、何を返すか」を書いているだけだ。プログラミング言語ではない。マークダウンで手順を書く。それだけ。

冒頭のフロントマター（ファイル先頭の設定欄）で、スキルの名前と説明、使えるツールを指定している。descriptionに「次のタスク」「次何する」のように使う場面を書いておくと、Claude Codeが会話の文脈から判断してスキルを起動する。`/search-task` とスラッシュコマンドで直接呼び出すこともできる。

「次のタスクは？」と聞くだけで、search-taskが動く。プロンプトを打つ必要がなくなった。

---

## 作り方

自分でゼロから書く必要はない。

Claude Codeに「こんなスキルが欲しい」と伝える。SKILL.mdを作ってくれる。保存して使うだけだ。

保存先は `~/.claude/skills/スキル名/SKILL.md`。全プロジェクト共通で使える。プロジェクト単位なら `.claude/skills/スキル名/SKILL.md`。

合わなければ直す。SKILL.mdはただのテキストファイルだ。

Anthropicが公式のスキル集を公開している。

https://github.com/anthropics/skills

参考になるが、探し回るよりAIを使って自分で作る方が早い。自分の作業に合わせたスキルは、自分にしか作れない。

---

## 育てる

search-taskはまず書いてみたスキルだ。作ったら便利だったので、そのまま使っている。

別のプロジェクトでは、TDDサイクル全体をスキル化した。冒頭で触れたtdd-skillsだ。

https://github.com/morodomi/tdd-skills

こっちは初期構想からだいぶ変わった。2025年12月にv1.0を公開したときは、TDDの7フェーズを回すだけだった。今はv5.1。何回もアップデートを重ねている。

Claude Codeがアップデートされるたびに、スキルも合わせて更新する。最近追加されたAgent Teams（複数エージェントの協調機能）で、レビューを討論型に作り替えた。RED/GREENフェーズの並列実行は、workerエージェントで回すようにした。

SKILL.mdを開いて、マークダウンを編集する。数分で終わる。その繰り返しで、自分のClaude Codeが出来上がっていく。

---

## まとめ

Claude Codeを使っているなら、スキルを作れ。

search-taskを作ってから、次に何をやるか悩む時間がなくなった。あの長いプロンプトが、「次のタスクは？」の一言で済む。

まず、今日使ったプロンプトを1つ選べ。それをSKILL.mdに書く。それで明日の作業が変わる。
