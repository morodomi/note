# コロナ禍で地方の中小企業がエンジニアを一気に複数名採用できた実話。そして2026年に同じ構造が再来しそうな話

2020年4月から、ある地方の中小企業グループのDX推進を週2日で伴走している。  
コーディングは一切せず、ただ組織の内製化をどう加速させるかだけを考えていた。

当時の情シスチームは  
・50代の正社員2名  
・派遣2名  
・新卒2名  

という構成だった。  
基幹システムの維持に毎年数千万円規模の費用がかかり、メンテナンス要員も不足気味で、「内製化なんて遠い話」という状態だった。

#### 2020年の奇跡のようなタイミング

ちょうどその年はコロナ禍の真っ只中。  
大手企業が新卒採用を大幅に絞ったタイミングだった。

すると、地元の情報系学部を卒業する学生から、  
例年とは明らかに違う規模の応募が集まった。  
結果、複数名を採用することになり、2021年入社でエンジニアチームは一気に7名体制になった。

50代の先輩社員はPCセットアップや運用保守をメインに回し、  
20代の若手7名で「レガシー基幹システムをモダンなクラウド環境へ移行する」プロジェクトを完遂した。  
移行後は保守費用が劇的に下がり、チームの自走度も明らかに上がった。

#### その後の推移

その後は年に数名ペースで採用を続けられたが、  
明らかに2021年がピークだった。  
ここ数年は「採用できない」状態が続いている。

#### 2026年の今、感じていること

大手企業がAIを理由にジュニア採用を絞り始めている動きは、  
2020年のコロナ禍と構造が非常に似ている。

一方で、今年の新卒は  
GitHub CopilotやClaude Code、Cursorなどを日常的に使いこなす「AIネイティブ」世代だ。

もし地方の中小企業が  
・AI開発ツールを完全に使い放題にする  
・週1回の技術顧問（CTO相当）を入れる  

という環境を整えれば、  
「大手には行けなかったけど、モダンな環境で早く実務を積みたい」  
という層が、再び集まる可能性は十分にあると思っている。

実際、X（旧Twitter）を見ていても、  
「AI時代はジュニア不要」「シニアだけ集めればいい」といった声が目立つ一方で、  
地方の中小企業向けの採用相談も増えている。

#### 最後に

3年前に起きた「採用の奇跡」は、決して偶然ではなかった。  
タイミングと環境が揃えば、地方の中小企業でも十分に優秀なエンジニアを集め、内製化を加速できる。

そして2026〜2027年卒は、また同じタイミングが来ている気がしてならない。

技術顧問として現場に入っている者として、  
この波に乗る準備を今から始めておくべきだと、本気で思っている。
