# AIが全部書く時代に、技術的ブレイクスルーはどこから来るのか

## 角度

AIコーディングでワークフローが確立 → レビュー文化が消える → 技術的難易度より
ビジネスインパクトが重視される → 新しいアルゴリズムは誰が生むのか？

## 思考の連鎖

1. レビュー文化の消失: 仕様書・コード・テストのレビューが物理的に不可能に
2. 評価軸の変化: 「技術的にどう難しいか」→「ビジネスインパクトがあるか」
3. ブレイクスルーの源泉: アイデア＝既存知識の組み合わせ。その組み合わせ探索もAIになるなら、人間の役割は何か？
4. 問い: 技術的ブレイクスルーは今後も人間から生まれるのか、それともAIの組み合わせ探索から生まれるのか

## 注意

- 実体験が弱い。思考実験寄り
- exspec開発や自身のAI開発経験と接続すると地に足がつく
- 候補#1「レビューできない時代」と接続可能
