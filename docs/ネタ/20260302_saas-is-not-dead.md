# SaaS is Deadは嘘。AIの本質は「捨てられる」ことにある

## きっかけ
- 自作スキルを捨てた記事を書いていて、同じ原則がSaaSにも当てはまると気づいた

## 立場
- SaaSの現場で働いているわけではない
- Claude Codeを使う開発者としての視点。自作ツールを作っては捨てる経験から語る
- 現場からの意見として「SaaS is not Dead」

## 主張
- AIの本質は捨てられるところにある
- 「SaaS is Dead」は間違い。AIの台頭でSaaSはもっと伸びる
- 理由: メンテナンス、情報の整合性を考えたら、SaaSに集約すべき
- 個人利用はMCP化して、SaaSのデータを10倍扱うようになる
- ただし、AI活用できるかどうかはどのSaaSにも掛かっている

## 論点
- SaaS = バラバラの現場データをフォーマットに落とし込む作業
- DX推進するほどSaaS化したほうがいい
- 自作SaaSを作る → 既存SaaSにデータ移行 → 自作SaaSは捨てる
- いつまでもメンテナンスするもんじゃない。事業に集中しろ
