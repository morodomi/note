## ブラッシュアップメモ（2026-02-16）

### 文体の問題

- 煽り文体+架空シナリオになっている
- 既存の公開記事は冷静な一人称エッセイ調で、一次情報ベース
- PVが高い記事（確定申告655PV、スキル77PV、TDD84PV）は全部「実際にやったこと」の記録
- 「俺は部下だ」路線は実際にやっていないことを書くことになる

### 実際の活動とのズレ

- 「主従関係を宣言した」というストーリーだが、実際にやったのは構造的な制約の設計
- tdd-skills、CLAUDE.md、Subagent/Agent Teams、quality-gate、plan-review、socrates protocolなど、仕組みで制御している
- 「部下だと言ったから賢くなった」ではなく「レールを敷いたから暴走しなくなった」が事実

### 既出テーマとの重複

- 「AIに首輪はつけられない。だからレールを敷いて、セキュリティゲートも置いた」(1/26)
- 「Claude Codeが暴走しなくなった。tdd-skillsで1チケットを処理する流れ」(2/10)
- このまま書くと焼き直しになる

### 対案

- 2/16にdev-crewにdrift detection（pre-commit hook）を追加した
- 「AIが設計から逸脱してないかを、AIが自動で検出する」仕組みの話に寄せれば未発表の一次情報になる
- 切り口: AIの暴走を「事後レビュー」から「事前検出」に変えた実践記録
